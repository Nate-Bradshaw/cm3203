An optimisation problem instance: a pair of (F,c), where:
F - any set (set of feasible solutions) - set of bars that exist that could follow, restricted by the rules imposed by MIDI alongside my own restrictions
c - cost function, mapping c: F -> R(real numbers?)

if c(f) <= c(y) (cost of solution f <= cost of solution y) for all y ∈ F then f is globally optimal, there may be (probs will be) local optima that the program will find

Genetic Algorithms:

Chromosome: string of "genes" representing a solution, each gene represents the value of a parameter in the solution set

What we need to do is find a way to represent notes for the GA

Population: set of chromosomes (aka possible solutions)

Genetic operators: Selection, Crossover and Mutation

Selection: the act of selecting two "parents" for crossover. based on the fitness. The higher the fitness the more often that parent should be chosen

Crossover: with two parents selected, a crossover point is chosen. two children are made via switching chromosomes after the crossover point

if the crossover is randomly chosen and then can thus cut up notes, then the system will be capable of creating different note lengths from the parents (starting with a random population of quarter notes would be able to evolve into 1/8th and 1/16th ect)

crossover point could also have a chance to include both notes, thus creating a chord?

also using this, we won't get any notes longer than a quarter, so some way of tie-ing notes needs to be included

Mutation: with a small probability, change the value of some chromosome

mutation could also add another note in the place of a note to create chords
